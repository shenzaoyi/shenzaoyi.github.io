Lecture 0

三大抽象和三大编程范式

行为抽象（Action Abstraction）

  • 例如用函数封装一段操作，就是“行为抽象”。
  • add(x, y) 把“加法”这个行为变成了一个可以复用的抽象。

数据抽象（Data Abstraction）

  • 用结构体/类来封装数据本身。
  • 比如：class Student { name, id }，将数据捆绑起来，屏蔽内部结构。

控制抽象

面向对象编程范式（OOP）

  • 抽象的单元是“对象”——它把数据和行为合成在一起。
  • 对象 ≈ 抽象的角色 + 对应的行为（方法）

命令式编程范式

  • 常见于C语言，程序是“命令流水线”，一步步操作内存和状态。

函数式编程范式

  • 把行为本身作为参数传递。数据是不可变的，函数是“值”。
  • Scheme就是函数式语言的代表。
