parenting system

the qt class hierarchy makes many things convient:

  • When an object is destroyed, all of its children are destroyed as well. So, calling delete becomes optional in certain cases.
  • All QObjects have findChild and findChildren methods that can be used to search for children of a given object.
  • Child widgets in a QWidget automatically appear inside the parent widget.

QObject::connect 用来链接信号和槽，原型如下

参数解释： signals and slots

sender

  • 类型：const QObject *
  • 含义：发出信号的对象，也就是“信号源”。

signal

  • 类型：函数指针，形如 &SenderClass::someSignal
  • 含义：你要连接的信号标识符。它必须是 sender 类里的一个 signals: 中声明的信号名。

receiver

  • 类型：const QObject *
  • 含义：接收信号并执行槽函数的对象。
  • 注意：它同时也是上下文对象（context），决定槽函数执行所在的线程。若信号跨线程发射，Qt 会根据此对象的线程自动选择直接调用还是排队调用。

slot

  • 类型：函数指针，形如 &ReceiverClass::someSlot，也可以是任意可调用对象（例如 C++11 的 lambda）
  • 含义：当 signal 发射时，Qt 会自动调用这里指定的槽函数或可调用对象。参数类型要与信号匹配（或可隐式转换）。

type （可选）

  • 类型：Qt::ConnectionType 枚举
  • 默认：Qt::AutoConnection
  • 含义：指定调用方式——
    • Qt::DirectConnection：发射信号的线程直接调用槽（就像普通函数调用）。
    • Qt::QueuedConnection：将调用封装为事件，排入 receiver 所在线程的事件队列，异步执行。
    • Qt::AutoConnection（默认）：同线程用直接，跨线程用排队

qt ui

设计 QT 的 GUI 有两种方法， 第一种自己创建每个控件，并赋予属性；第二种就是使用 Qt Designer, 生成.ui 文件。

使用 Qt Designer 生成的 .ui 文件，通过 UIC (UI complier) 自动生成一个 c++ 类，包含

  • .ui 文件中的所有控件
  • setupUi() 函数， 负责将界面控件和当前窗口绑定
